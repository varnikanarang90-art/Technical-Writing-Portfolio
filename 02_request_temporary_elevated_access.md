# Requesting Temporary Elevated Access in Pathlock  
*A Sample Task-Based Guide*

## Purpose  
This guide explains how to request temporary elevated access in Pathlock. Elevated access is used for short-term tasks such as emergency fixes, troubleshooting, or administrative activities.

## When to Request Elevated Access  
- To resolve a production issue  
- To perform an urgent administrative task  
- To complete a critical operation that requires temporary permission

---

## Steps  
1. Log in to Pathlock.  
2. Go to **Access Requests**.  
3. Select **New Request**.  
4. Choose **Temporary / Elevated Access**.  
5. Select the application (SAP, Oracle, Workday, etc.)  
6. Choose the required role or permission.  
7. Set the **duration** (start and end time).  
8. Enter a clear **business justification**.  
9. Submit the request for approval.

---

## After Approval  
- Access is granted only for the selected time window.  
- All activity during the elevated access session is logged.  
- Access automatically expires after the set duration.

---

## Tips  
- Request only the level of access needed.  
- Keep your justification short and clear.
